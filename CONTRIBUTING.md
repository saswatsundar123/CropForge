# Contributing to CropForge

Thank you for your interest in contributing!

## Code Style
- Python 3.12+ syntax
- Type hints on all public functions
- Docstrings (NumPy style) on all public classes and methods

## Testing
- Every new class or function must have a corresponding test in `tests/`
- Run `pytest` before opening a PR; all tests must pass

## Opening Issues
Use the GitHub Issues labels: `bug`, `enhancement`, `crop-package`, `documentation`

## Pull Requests
Standard fork-and-PR workflow. Reference the related issue in the PR description.
